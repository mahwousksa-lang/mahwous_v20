# 🧪 مهووس v20.5 — نظام التسعير الذكي بذاكرة عبقرية

## 🧠 المنظومة الذكية المتكاملة

### المشكلة 1: المفقود موجود عندي باسم آخر!
```
المنافس: "ديور سوفاج بارفيوم 100 مل"
عندك:    "Dior Sauvage EDP 100ml"
→ النظام يكتشف التشابه (60%) ويسألك: "هل هذا نفسه؟"
→ تضغط ✅ نعم → يتعلم ولن يسألك مرة ثانية
→ تضغط ❌ لا → يبقى مفقود فعلاً
```

### المشكلة 2: البيانات تختفي عند رفع ملفات جديدة!
```
قاعدة بيانات تراكمية (11 جدول):
├── كل سعر يُضاف للتاريخ (لا يُحذف أبداً)
├── المنتج المُدخل سابقاً → يُحدَّث السعر فقط
├── المفقود المسجل سابقاً → يُحدَّث عدد المنافسين
└── كل تحليل جديد يُضاف للذاكرة السابقة
```

### المشكلة 3: كيف أعرف ماذا عدّلت سابقاً؟
```
نظام ترحيل التعديلات:
├── معلق (pending) → تم إنشاء التعديل
├── مُرسل (sent) → أرسلته للمتجر
├── مؤكد (confirmed) → تم تطبيقه في المتجر
└── كل خطوة مسجلة في سجل الإجراءات
```

### المشكلة 4: المفقودات المُرحّلة
```
دورة حياة المفقود:
├── new → مفقود جديد
├── needs_review → يشبه منتج عندك (يحتاج قرارك)
├── is_duplicate → أكدت إنه موجود عندك باسم آخر (تعلّم النظام)
├── verified_missing → أكدت إنه مفقود فعلاً
├── added_to_store → تم ترحيله وإضافته في المتجر
└── ignored → لا تريده
```
